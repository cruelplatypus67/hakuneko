## Quick start

[source,bash]
----
git clone https://github.com/cruelplatypus67/hakuneko.git
cd hakuneko
npm install
----

=== Development

[source,bash]
----
# Terminal 1 — host the web app with hot reload
npm run serve:dev --workspace web

# Terminal 2 — desktop shell pointed at localhost:3000
npm run launch:dev --workspace app/electron
# or: npm run launch:dev --workspace app/nw
----

=== Checks and tests

[source,bash]
----
npm run check          # TypeScript, ESLint, Svelte/Vue checks
npm run test           # Unit / component tests
npm run test:e2e       # End-to-end (builds workspaces first)
----

=== Package desktop builds (macOS host)

Builds a **Windows x64** portable ZIP and a **macOS arm64** DMG into `dist/`, then removes intermediate Electron redistributables.

[source,bash]
----
npm run package:win-mac
# equivalent: node scripts/package-win-mac.mjs
----

Options:

* `--skip-install` — skip `npm install`
* `--no-cleanup` — keep temp redists and `app/electron/build`

NOTE: Off-Windows packaging skips `rcedit` icon/version stamping (Windows-only tool). The ZIP remains runnable as `hakuneko-electron.exe`.

## Security review notes

A source review was performed on this tree for hidden malware-style behavior (install hooks, obfuscated C2, credential theft, miners, unexpected outbound exfiltration). Summary:

=== What looks intentional / expected for this app

* **Scraping desktop shell** — Electron sets `webSecurity: false` to bypass CORS for site fetches. The main renderer is sandboxed with Node integration off, context isolation on, an allowlisted IPC bridge, exact sender checks, and restricted navigation/permissions (`app/electron/src/Main.ts`).
* **Web app origins** — Electron and NW.js package the locally built web UI. NW.js no longer grants Node access to remote origins. Electron accepts an explicit `--origin` override for local development only.
* **Obfuscated modules** (high scrutiny, domain-specific purpose):
  ** `web/src/engine/platform/AntiScrapingDetection.js` — bot / challenge detection helpers
  ** `web/src/engine/websites/*.DRM.js` — site-specific image/DRM descramblers (e.g. Bilibili, Comikey, MangaGo)
  These are heavily obfuscated (string encoding + control-flow packing). Readable exports match downloader use (e.g. `DRMProvider`, fetch redirection enums), not a general reverse shell. Treat them as **opaque third-party-style blobs**: re-audit on every update.
* **Ad / tracker blocking** — `BloatGuard` blocks many ad and analytics hosts (including Google Analytics and Sentry), which reduces drive-by noise from manga sites.
* **Local RPC** — Optional websocket RPC (`app/src/rpc/Server.ts`) binds only to loopback and is gated by an origin allowlist plus a SHA-256 token derived from a randomly generated user secret.
* **`postinstall`** — Only runs Electron once to print its version (`--no-sandbox --version`). No download-and-execute of unrelated payloads observed.

=== Risks and weaknesses (not “hidden malware”, but real)

[cols="1,3",options="header"]
|===
| Risk | Detail

| Trust of hosted app
| Packaged desktop clients use a local UI, reducing this risk. The hosted web deployment remains in scope for normal browser data and session compromise, and explicit desktop `--origin` development overrides should only point to trusted servers.

| Obfuscated DRM / anti-bot JS
| Hard to manually audit. Prefer pinning known-good revisions; avoid running untrusted forks of these files.

| GitHub-sourced npm deps
| `pdfkit` and `websocket-rpc` are pulled via `github:manga-download/...` pins. Supply-chain trust sits with that org and those tags.

| Weak secret storage
| Settings secrets use Base64 (`btoa` / `atob`), not real encryption (`web/src/engine/SettingsManager.ts`). Local disk access can recover stored passwords.

| `webSecurity: false`
| Expected for the product, but widens renderer exposure if a site XSS succeeds inside the shell.

| Dead telemetry stub
| `WebtoonHatti.ts` contains a **commented-out** reporter to `report.hakuneko.workers.dev`. Not active, but shows prior telemetry design.
|===

=== Not found

No evidence of classic malware patterns in first-party sources: reverse shells, keyloggers, crypto miners, Discord/Telegram webhook exfiltration, or install scripts that pull and run remote binaries outside Electron’s normal install.

IMPORTANT: This is a best-effort static review of the repository sources, not a formal penetration test or full deobfuscation of every DRM blob. Re-run checks after dependency or DRM file updates.
